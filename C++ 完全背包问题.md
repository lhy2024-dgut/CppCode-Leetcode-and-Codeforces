### 💻 架构师专属：完全背包通用模板 (只需改动一行！)

```cpp
int unboundedKnapsack(int W, const vector<int>& wt, const vector<int>& val) {
    int n = wt.size();
    vector<int> dp(W + 1, 0);

    for (int i = 0; i < n; i++) {
        // ⚠️ 核心巨变：正序遍历！(从 wt[i] 一路向右遍历到 W)
        // 因为是正序，算 dp[j] 的时候，查的 dp[j - wt[i]] 是【在当前这一轮刚刚被更新过的值】。
        // 这就完美实现了“装了这件物品后，只要容量够，我还能继续装它”的业务逻辑！
        for (int j = wt[i]; j <= W; j++) {
            dp[j] = max(dp[j], dp[j - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

```

**1. 力扣 416：分割等和子集 (Partition Equal Subset Sum)**

* **业务马甲：** 给您一个只包含正整数的数组，问能不能把这个数组分成两个子集，使得两个子集的元素和相等。
* **架构师拆解：** 这题简直绝了！如果能平分，说明总和一定是偶数。那么业务目标就变成了：**“我能不能从数组里挑出几个数字，让它们的总和刚好等于 `总和 / 2` ？”** * **映射背包：** 背包容量就是 `总和 / 2`。每个数字既是重量 `wt`，也是价值 `val`。直接套 01 背包逆序模板，看最后 `dp[总和/2]` 是不是恰好等于 `总和/2`。

**2. 力扣 494：目标和 (Target Sum)**

* **业务马甲：** 给您一个数组，您可以在每个数字前面加 `+` 或者 `-`，问有多少种不同的方法，能让最终结果等于 `target`。
* **架构师拆解：** 假设加 `+` 的数字总和是 $A$，加 `-` 的数字总和是 $B$。那么 $A + B = sum$，$A - B = target$。两式相加除以 2，推导出 $A = (sum + target) / 2$。
* **映射背包：** 问题瞬间降维成：**“在数组里找几个数，让它们的和恰好等于 A 的方案数是多少？”** 典型的求组合数的 01 背包！

#### 🗡️ 第二战区：完全背包（每件物品能用无数次）

**3. 力扣 322：零钱兑换 (Coin Change)** —— **【全网必刷经典】**

* **业务马甲：** 给您几种不同面值的硬币（每种硬币无限个），和一个总金额。问凑成这个总金额**最少**需要几枚硬币？
* **架构师拆解：** 既然硬币无限个，必是完全背包。注意，这里求的不是“最大价值”，而是“最少数量”。
* **映射背包：** 背包容量是总金额。硬币面值是重量，价值统统算作 1（代表一枚）。状态转移方程里的 `max` 要改成 `min`，也就是 `dp[j] = min(dp[j], dp[j - coins[i]] + 1)`。

**4. 力扣 518：零钱兑换 II (Coin Change II)**

* **业务马甲：** 还是无限硬币，还是凑总金额。但这题问的是：**总共有多少种凑法**？
* **架构师拆解：** 这是完全背包求“方案总数”的终极形态。
* **映射背包：** 正序遍历模板。转移方程变成累加：`dp[j] = dp[j] + dp[j - coins[i]]`（我不拿这枚硬币的方案数 + 我拿这枚硬币的方案数）。
